# Curso de Git, GitHub e SourceTree

Este repositório foi criado como registro do conhecimento adquirido após a conclusão de **dois cursos introdutórios sobre Git, GitHub e controle de versão**.

O objetivo é demonstrar os principais conceitos aprendidos e como eles são utilizados no desenvolvimento de software moderno.

---

# Principais conceitos aprendidos

## Git
O **Git** é um sistema de **controle de versão** que permite registrar alterações em arquivos ao longo do tempo, facilitando o acompanhamento do histórico do projeto e o trabalho em equipe.

Durante o curso aprendi:

- Criar repositórios com `git init`
- Adicionar arquivos com `git add`
- Registrar mudanças com `git commit`
- Enviar alterações com `git push`
- Atualizar projetos com `git pull`
- Entender o funcionamento do **HEAD**

---

## Uso do Terminal

Também foram abordados comandos básicos do terminal, como:

- `cd` → navegar entre diretórios
- `mkdir` → criar pastas
- remoção de diretórios
- uso do símbolo `~` para representar o diretório inicial do usuário

---

## Branches

Aprendi a utilizar **branches**, que permitem criar diferentes linhas de desenvolvimento dentro do mesmo projeto.

Isso possibilita:

- desenvolver novas funcionalidades sem alterar o código principal
- trabalhar em diferentes tarefas ao mesmo tempo
- fazer **merge** entre branches
- organizar o histórico de commits

---

## GitHub

O **GitHub** é uma plataforma para **hospedagem de repositórios Git**, permitindo armazenar projetos online e colaborar com outros desenvolvedores.

Principais conceitos estudados:

- criação de repositórios
- uso de **remote origin**
- envio de código para o repositório remoto
- **fork** de projetos
- repositórios públicos e privados

---

## SourceTree

O **SourceTree** é uma ferramenta que fornece uma **interface gráfica para utilizar o Git**.

Com ele é possível:

- visualizar commits e branches
- gerenciar repositórios de forma visual
- realizar push e pull sem usar apenas o terminal
- facilitar o entendimento do histórico do projeto

---

## Git Flow

Também foi estudado o **Git Flow**, um modelo de organização de branches que define uma estrutura de desenvolvimento.

Branches utilizadas nesse modelo:

- `main`
- `develop`
- `feature`
- `release`
- `hotfix`

---

# Conclusão

Após a conclusão dos cursos, adquiri conhecimentos fundamentais sobre **versionamento de código, organização de projetos e colaboração utilizando Git e GitHub**.

Essas habilidades são essenciais para o desenvolvimento de software e para o trabalho em equipe na área de tecnologia.
